# Logging Guide

このガイドは、朝ランニングの記録を無理なく続けるためのルールです。

## 基本方針

記録の目的は、完璧な日誌を作ることではありません。

目的は、朝ランニングを通じて得た体調・思考・習慣の変化を、後から使える形で残すことです。

## 毎日書くべき最小項目

忙しい日は、以下の3つだけで十分です。

```md
## Run Summary
- Distance:
- Time:
- Condition:

## Morning Insights
-

## One thing to act on today
-
```

## 音声メモから整理する場合

走りながらの音声入力は、多少カオスでも問題ありません。

整理するときは、次の4分類に分けます。

1. ランニングの内容
2. 身体の状態
3. 仕事・生活への気づき
4. 今日やる小さなアクション

## ファイル名ルール

```text
logs/YYYY/YYYY-MM-DD.md
```

例:

```text
logs/2026/2026-05-24.md
```

## よい記録の条件

- 事実がある
- 感覚がある
- 次の行動がある
- 後から読み返して、自分の状態が分かる

## 週次レビューの観点

週末に、以下を確認します。

- 距離や時間は増えすぎていないか
- 疲労や痛みのサインはないか
- 朝ラン後の集中力は上がっているか
- 繰り返し出てくる思考テーマは何か
- 来週、何を少しだけ改善するか

## メンテナンス思想

ランニングログは、人生の生産技術ノートです。

工場でいう日報・改善記録と同じく、観察、仮説、実行、振り返りを回すための素材になります。
